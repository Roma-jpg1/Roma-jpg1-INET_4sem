# Вопросы для самопроверки

1. В чем разница между Forward Proxy (обычный прокси) и Reverse Proxy (обратный прокси)?

Forward Proxy — стоит перед клиентом, скрывает клиента.
Reverse Proxy — стоит перед серверами, скрывает backend (как Nginx в задании)

2. Что такое Upstream в терминологии Nginx?

Upstream — это backend-сервис, куда Nginx пересылает запрос.

3. Зачем скрывать реальные IP-адреса сервисов от клиента?

Скрывают IP для безопасности, изоляции внутренней сети и возможности менять архитектуру без изменения клиента

4. Какие еще задачи, кроме маршрутизации, может выполнять API Gateway (назовите хотя бы 2)?

аутентификацию
балансировка нагрузки 

5. Что произойдет, если Gateway упадет? Как можно от этого защититься?

сервис станет недоступным, для защиты: репликация, балансировщик перед gateway
